# API Changelog — 2026-04-18

## Breaking Changes

### Icon is now a URL (all roles)

Profile icons are no longer returned as base64 strings. The `icon` field is now a **public GCS URL** (e.g. `https://storage.googleapis.com/my-saga-profiles/user/1`) or `null`.

Affects:
- `POST /user/dashboard`
- `POST /organizer/dashboard`
- `POST /boss/dashboard`
- `POST /user/update-profile` (response)
- `POST /organizer/update-profile` (response)
- `POST /boss/update-profile` (response)
- `POST /search/profile`

**Frontend change required:** Replace `data:image/png;base64,${icon}` with just `src={icon}` directly.

### Sending icons for update-profile (all roles)

No change on the request side — continue sending raw base64. The backend now uploads it to GCS and stores the URL.

---

### User dashboard — `star_score` removed

`POST /user/dashboard` no longer returns `star_score`. Replaced with 6 index fields:

| Field | Type | Default |
|-------|------|---------|
| `cognitive_index` | float | 50 |
| `drive_index` | float | 50 |
| `adaptability_index` | float | 50 |
| `integrity_index` | float | 50 |
| `emotional_intellect_index` | float | 50 |
| `creativity_index` | float | 50 |

---

## New Fields

### Organizer dashboard — `rating`

`POST /organizer/dashboard` now returns `rating` (float, default 3.9).

---

## Bug Fixes

- `POST /boss/update-profile`, `POST /organizer/update-profile`, `POST /user/update-profile` — response now correctly returns the updated icon instead of a corrupted Buffer object.
